# A포털 운영 업무 매뉴얼

## 1. 업무 운영 및 보안 원칙

### 1.1 운영 원칙
- A포털은 웹 기반 포털 서비스로 데이터 조회, 저장, 분석 기능을 제공
- 기본 서비스는 무료 제공, 추가 기능은 별도 요금 부과 정책 적용
- 유연 근무제를 통한 업무 효율성 확보

### 1.2 보안 원칙
- **비밀번호 정책**: 최소 8자 이상, 90일 주기 변경, 재사용 금지
- **접근 통제**: 업무 최소 권한 원칙, 퇴사자 계정 즉시 비활성화
- **데이터 보호**: 중요 데이터 암호화 저장, 외부 반출 시 사전 승인 필수
- **네트워크 보안**: 공용 Wi-Fi 사용 금지, VPN 사용 권장
- **보안 사고 대응**: 발생 즉시 보안팀(security@company.com)에 보고

## 2. 주기별 업무 목록

### 2.1 일상 업무
- **API 운영 관리**: 서비스 API 상태 모니터링 및 성능 점검
- **데이터 파이프라인 점검**: 데이터 처리 프로세스 정상 작동 확인
- **보안 체크**: 접근 로그 및 보안 이벤트 점검

### 2.2 주기별 업무
- **주간**: 시스템 성능 분석 및 개선 포인트 식별
- **월간**: 서비스 이용 현황 분석 및 보고서 작성
- **분기**: 보안 정책 준수 현황 점검 및 개선 방안 수립
- **연간**: 서비스 계약 갱신 검토 (계약 기간: 1년)

## 3. 표준 실무 가이드

### 3.1 채용 및 인력 관리
**Why**: 서비스 운영 및 개발을 위한 전문 인력 확보
**Where**: 백엔드 개발자, 데이터 엔지니어 포지션 중심
**How**: 
- 채용 공고 게시 (recruit@aportals.com 접수)
- 필수 요건: Python/Java 개발 경험, REST API 설계 경험, Git 사용 경험
- 우대 사항: 클라우드(AWS, GCP) 경험, 대용량 데이터 처리 경험
**Detail**: 정규직 채용, 서울 근무, 유연 근무제 적용

### 3.2 서비스 계약 관리
**Why**: 고객과의 명확한 권리·의무 관계 설정
**Where**: 온라인 플랫폼 서비스 제공 범위 내
**How**:
- 계약서 체결 (1년 단위)
- 기본 서비스 무료 제공, 추가 기능 유료화
- 고객 해지 요청 시 언제든 처리 가능
**Detail**: 2026년 1월 1일부터 효력 발생 계약 기준

### 3.3 보안 관리 프로세스
**Why**: 회사 정보 보호 및 법적 요구사항 준수
**Where**: 전사 공통 적용
**How**:
- 비밀번호 정책 시행 (8자 이상, 90일 변경)
- 접근 권한 최소화 원칙 적용
- 데이터 암호화 및 외부 반출 통제
**Detail**: 보안 사고 발생 시 즉시 security@company.com 보고
