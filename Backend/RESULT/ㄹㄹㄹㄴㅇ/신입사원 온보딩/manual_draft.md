# 신입사원 온보딩 업무 매뉴얼

## 1. 신입사원을 위한 기본 보안 원칙

### 1.1 필수 보안 수칙
신입사원으로서 반드시 알아야 할 회사 정보 보호 규정입니다.

**내 비밀번호 관리법**
- 최소 8자 이상으로 설정하세요
- 90일마다 새로운 비밀번호로 변경하세요
- 이전에 사용한 비밀번호는 재사용하지 마세요
- 개인 계정과 회사 계정의 비밀번호를 다르게 설정하세요

**업무용 계정 사용 원칙**
- 본인에게 부여된 계정만 사용하세요
- 동료와 계정을 공유하지 마세요
- 업무에 필요한 시스템에만 접근하세요

**회사 데이터 보호하기**
- 중요한 회사 문서는 개인 이메일로 전송하지 마세요
- USB나 외부 저장장치 사용 전 팀장에게 문의하세요
- 회사 노트북에 개인 파일을 저장하지 마세요

**안전한 네트워크 사용**
- 카페나 공공장소의 Wi-Fi로 업무하지 마세요
- 재택근무 시 VPN을 반드시 연결하세요

### 1.2 경력별 보안 주의사항

#### 📚 신입사원 (1년차)
- 기본 보안 규칙을 철저히 익히고 습관화하기
- 모르는 것은 바로 멘토나 보안팀에 문의하기
- 보안 교육을 빠짐없이 수강하기

#### ⚡ 중급자 (2-3년차)
- 후배들에게 보안 원칙을 알려주고 모범 보이기
- 프로젝트 리딩 시 팀원들의 보안 준수 상황 점검하기
- 새로운 보안 위협에 대해 지속적으로 학습하기

#### 🚨 **경력자 (3년차 이상) 특별 주의사항**
**높은 권한으로 인한 위험 관리**
- 고급 시스템 접근 권한이 많아질수록 더욱 신중하게 행동하세요
- 운영 서버 접근 시 모든 작업을 반드시 로그로 남기세요
- 권한 관련 요청을 받았을 때는 2차 승인을 받고 처리하세요

**소셜 엔지니어링 대응**
- 경력이 쌓일수록 사회공학적 공격의 주요 타겟이 됩니다
- 전화나 메신저로 받는 급한 업무 요청에 특히 주의하세요
- 회사 외부에서 내부 정보를 묻는 연락이 오면 즉시 보안팀에 신고하세요

**멘토링 시 보안 책임**
- 후배들의 보안 의식 수준을 정기적으로 점검하세요
- 코드 리뷰 시 보안 취약점도 함께 확인하세요
- 팀 내 보안 문화 조성에 적극적으로 참여하세요

### 1.3 문제 상황 시 대응 방법
보안 관련 문제가 생겼을 때는 당황하지 말고 즉시 보안팀(`security@company.com`)에 연락하세요.

## 3. 신입사원 노트북 설정 가이드

### 3.1 왜 노트북 설정이 중요한가요?

#### 목적
- 회사 업무를 효율적으로 수행할 수 있는 표준 환경 구축
- 회사 보안 정책에 맞는 안전한 업무 환경 만들기
- 팀원들과 동일한 개발 환경으로 협업 효율성 높이기

#### 누가 해야 하나요?
- 모든 신입 개발자 (백엔드 개발자, 데이터 엔지니어)
- 회사에서 지급받은 노트북 사용자

### 3.2 단계별 설정 방법

**1단계: 기본 계정 설정하기**
```
✅ 해야 할 일:
1. 받은 회사 이메일로 로그인 테스트
2. Slack 가입하고 우리 팀 채널 들어가기
3. Jira에서 내 계정 만들기

💡 팁: 모든 계정의 비밀번호는 메모장에 적어두세요!
```

**2단계: 필수 프로그램 설치하기**
```
✅ 설치할 프로그램들:
1. Chrome 브라우저 (기본 업무용 브라우저)
2. VS Code (코드 작성용 에디터)
3. Docker (개발 환경 구축용)
4. Git (소스코드 관리용)

💡 팁: 설치 중 오류가 생기면 IT 지원팀에 바로 문의하세요!
```

**3단계: 개발 환경 준비하기**
```
✅ 개발 도구 설치:
1. Python 3.10 버전 설치 (반드시 이 버전!)
2. Node.js 최신 버전 설치
3. 환경 변수 설정 (멘토가 도와줄 예정)

💡 팁: Python 버전이 다르면 팀 프로젝트가 안 돌아갈 수 있어요!
```

**4단계: 보안 설정 완료하기**
```
✅ 보안 필수 설정:
1. 회사 지정 백신 프로그램 설치
2. 노트북 디스크 암호화 활성화

💡 팁: 보안 설정은 절대 건드리지 마세요. 문제 생기면 IT팀 연락!
```

**5단계: 코드 저장소 접근하기**
```
✅ GitHub 설정:
1. GitHub에서 회사 조직 초대 수락하기
2. SSH 키 만들어서 등록하기

💡 팁: SSH 키 만드는 방법 모르겠으면 선배 개발자에게 물어보세요!
```

### 3.3 설정 완료 후 확인사항
- Python 버전이 정확히 3.10인지 확인
- Node.js가 제대로 설치되었는지 확인
- 디스크 암호화가 켜져있는지 확인
- GitHub에서 회사 프로젝트가 보이는지 확인

## 4. 신입사원이 알아야 할 기술 역량

### 4.1 우리 회사에서 필요한 기술들

#### 백엔드 개발자라면
**반드시 알아야 할 것들:**
- Python 또는 Java로 프로그램 만들기
- REST API가 무엇인지 알고 만들 수 있기
- Git으로 소스코드 관리하기

**알면 더 좋은 것들:**
- AWS나 GCP 같은 클라우드 서비스 경험
- 많은 데이터를 빠르게 처리하는 방법

#### 데이터 엔지니어라면
**주요 업무:**
- 데이터가 잘 흘러가는 파이프라인 만들기
- 대용량 데이터 처리 시스템 구축하기

### 4.2 모르는 기술이 있다면?
- 멘토나 팀장에게 학습 계획에 대해 상의하세요
- 회사 내부 스터디 그룹 참여를 고려해보세요
- 온라인 강의 수강비를 회사에서 지원받을 수 있는지 문의하세요

## 6. 도움이 필요할 때 연락할 곳

### 6.1 상황별 연락처

| 어떤 문제인가요? | 누구에게 연락하나요? | 이메일 | 얼마나 빨리 답해주나요? |
|----------------|-------------------|--------|-------------------|
| 노트북이나 프로그램 문제 | IT 지원팀 | it-support@company.com | 하루 안에 |
| 보안 문제나 이상한 일 | 보안팀 | security@company.com | 바로 |
| 입사 관련 궁금한 것 | 인사팀 | recruit@aportals.com | 이틀 안에 |
| 업무나 기술 관련 질문 | 멘토 또는 팀장 | - | 바로 |

### 6.2 문제 상황별 대처법

**노트북 설정하다가 오류가 났어요!**
1. 오류 화면을 스크린샷으로 찍어두세요
2. 어떤 과정에서 오류가 났는지 메모하세요
3. IT 지원팀에 상세하게 설명해서 이메일 보내세요
4. 하루 정도 기다려보세요

**뭔가 이상한 메시지가 떴어요! (보안 관련)**
1. 당황하지 말고 일단 그 프로그램 사용을 멈추세요
2. 화면을 그대로 두고 보안팀에 즉시 전화하세요
3. 보안팀에서 지시하는 대로 따라하세요

**업무를 어떻게 해야 할지 모르겠어요**
1. 멘토나 팀장님께 부담 없이 질문하세요
2. 같은 질문을 여러 번 해도 괜찮으니 이해할 때까지 물어보세요
3. 질문 내용을 미리 정리해서 물어보면 더 좋아요

- 팀 회의나 스터디에 적극적으로 참여해보세요
